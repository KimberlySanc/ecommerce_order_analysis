# 📊 E-Commerce Data Analytics Project

## Overview

This project demonstrates a complete data analytics workflow from raw data to business insights. It includes data cleaning, exploratory data analysis (EDA), database querying using SQL, and creating an interactive dashboard.

The goal is to transform raw data into actionable insights through a structured, reproducible analytics pipeline.

---

## Dataset

* **Source:** Kaggle
* **Dataset:** E-commerce Orders Dataset 2026
* **Link:** https://www.kaggle.com/datasets/mmumairkhattak/e-commerce-orders-dataset-2026-scra

### Contents

The E-commerce Orders Dataset 2026 simulates a real-world online retail environment with **30,000 transaction-level records** and **41 structured features** spanning the full customer purchase lifecycle.

The dataset captures end-to-end e-commerce operations, including customer demographics, order activity, product-level attributes, pricing and discount strategies, payment methods, shipping logistics, customer engagement signals, review ratings, and profitability metrics.

## Project Steps

### 1. Data Cleaning & Preparation (Python)

* Loaded dataset using pandas
* Performed initial inspection using `.info()` and `.describe()`
* Checked and validated missing values
* Standardized column names to `snake_case` for consistency

#### Feature Engineering

* Created `age_group` by binning customer age ranges
* Created `customer_lifetime_value_tier` based on lifetime value segmentation
* Created `gross_sales_amount` using `unit_price × purchase_quantity`

#### Database Integration

* Connected Python environment to PostgreSQL
* Loaded cleaned dataset into the database for structured querying

### 2. Exploratory Data Analysis (SQL – PostgreSQL)

Key business queries included:

* Revenue by gender comparison
* Membership tier performance (average order value, revenue, transactions)
* Top 3 product subcategories per category
* Revenue and profit by age group
* Top traffic sources by age group
* High-spending discount user analysis
* Top 5 products by rating
* Shipping method performance comparison
* Discount dependency analysis by subcategory
* Customer lifetime value segmentation analysis

## Key Results & Insights

* Identified high-performing customer segments based on age group, membership tier, and traffic source.
* Found significant revenue differences across membership levels, highlighting subscription-driven value.
* Discovered top-performing product categories and subcategories.
* Determined the highest revenue-generating marketing channels for each customer age segment.
* Observed seasonal revenue patterns and fluctuations in customer purchasing behavior over time.

---

## Business Recommendations

* **Strengthen Membership Programs:** Expand benefits across tiers to increase retention and average order value.
* **Targeted Marketing by Age Group:** Align campaigns with top-performing traffic sources per demographic segment.
* **Tier Upgrade Strategy:** Encourage upgrades through personalized incentives for high-value customers.
* **Product Promotion Strategy:** Highlight top-rated products in marketing campaigns.
* **Discount Optimization:** Reduce over-reliance on discounts through loyalty programs and bundle offers.
* **High-Value Discount Segment:** Create targeted campaigns for customers who use discounts but still generate high spending.

## How to Run This Project

### 1. Clone the Repository

```bash
git clone https://github.com/KimberlySanc/ecommerce_order_analysis
cd ecommerce_order_analysis
```

### 2. Install Required Libraries

```bash
pip install pandas numpy psycopg2 sqlalchemy
```

### 3. Run Data Cleaning & EDA

```bash
jupyter notebook
```

Run all notebook cells in order to clean and transform the dataset.

### 4. Set Up PostgreSQL Database

* Create a new PostgreSQL database.
* Run the Python script to load cleaned data into SQL.
* Execute SQL queries from the `/sql` folder.

### 5. Open Power BI Dashboard

* Open the `.pbix` file in Power BI Desktop.
* Refresh the data source if necessary.
